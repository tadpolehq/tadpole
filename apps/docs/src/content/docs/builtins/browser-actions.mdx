This page defines all of the built in browser scoped actions.

## `new_page`
Opens a new browser tab (Session). All nodes within the new_page block are executed sequentially within that specific tab's context.

### Scope
Transitions from Browser Scope to Session Scope.

### Usage
```kdl
new_page {
  goto "https://google.com"
  type "Hello World!"
  $ "button[title='Search']" {
    click
  }
}
```

## `log`
Logs a message with the provided level.

### Args
- `message`: The message to log.

### Options
- `level`: The log level, can be `debug`, `info`, `warn` or `error`. Defaults to `info`.

### Usage
```kdl
log "Some log message"
log "Error!" level="error"
```

<Aside type="note">
The minimum log level is controlled in the cli using the `--log-level` option.
</Aside>

## `parallel`
Executes multiple Browser Actions concurrently. This is the primary way to perform multi-tab automation or run simultaneous workflows.
If any action fails, the entire operation will fail.

### Usage
```kdl
parallel {
  new_page { ... }
  new_page { ... }
}
```

<Aside type="note">
The parallel block will wait for all child actions to complete before the script moves to the next node in the main block.
</Aside>

## `sleep`
Pauses the execution of the current scope for a specified duration.

### Args
- `time`: The duration in milliseconds. Can be a static number or a dynamic expression.

### Usage
```kdl
// Static
sleep 2500

// Dynamic
sleep "=sec(2.5)"
```
